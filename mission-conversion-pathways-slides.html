Proposed Solution

Three pillars, different problems

No single change fixes both TOFU and BOFU. Each pillar targets a specific segment.

Primarily TOFU

Pillar 1: Nonlinear Structure

  • All challenges visible from start
  • User-chosen completion order
  • Visual progression tracker
  • Soft recommendations, no hard gates
Why: Users hit a hard sequential gate (AL) and drop off. Nonlinear lets them warm up before committing.
Primarily BOFU

Pillar 2: Progressive Challenges & Rotation

  • Progressive partner asks (500 → 1,500 → 3,000 RUB)
  • Monthly challenge rotation
  • Challenge template library per tier
  • Partner-approved variant ranges
Why: Same stale ask × 3 months = declining conversion. New asks + lower entry points give reasons to re-engage.
TOFU + BOFU

Pillar 3: Reward Rebalancing

  • Milestone rewards at progression thresholds
  • Exclusive profile items for partner tasks
  • Tiered reward escalation
  • Mystery Box on milestones (validated +61% AL)
Why: 83% spend FP on profile items. Exclusive rewards create incentive that gaming alone can't provide.
Technical note for Evugene: Pillar 2 requires a challenge template library — a backend data model storing partner-approved challenge variants per tier. The rotation scheduler auto-assigns templates monthly so no user sees the same partner ask two months running. MVP approach: manual template assignment by Activation Ops, automated scheduling in Phase 2.

Commercial Ask

What we need from you

  • 1
    Winline: Approve progressive deposit ladder
    Current: flat 3,000 RUB (FTD) / 5,000 RUB (WHALE). Proposed: 500 → 1,500 → 3,000 for FTD, 1,000 → 3,000 → 5,000 for WHALE. We need Winline's sign-off on acceptable deposit/bet ranges.
  • 2
    Winline: Define challenge variant library
    For monthly rotation, we need 3–4 approved challenge variants per tier. Can Winline approve a range (e.g., "deposits between 500–5,000 RUB") rather than individual challenges? This simplifies the workflow significantly.
  • 3
    Tradeit: MVP partner for Pillar 1
    Tradeit has the most diverse geos and implicit AL (lower-risk). We'd like to run nonlinear structure as the first A/B test on Tradeit missions. Need Commercial alignment on timing & reporting.
  • 4
    All partners: Confirm reporting requirements
    Challenge rotation changes the month-over-month comparability of partner reports. We'll tag challenges by tier & action type for consistent aggregation. Any partner-specific reporting needs?

Timeline ask: We need Winline variant approval before we can build the challenge template library. Ideal: start conversations this sprint, aim for approved variant ranges within 2 weeks.

Next Steps

Phased rollout plan

  • 1
    Nonlinear Structure + Rewards (Pillar 1 + 3) — Tradeit MVP
    Ship nonlinear mission structure with milestone rewards on Tradeit. A/B test vs sequential. Measure: partner task completion rate, AL rate, reward claim rate.
  • 2
    Progressive Challenges + Rotation (Pillar 2) — Winline
    Once variant ranges are approved, build challenge template library and launch progressive partner asks. Target the 43K conscious BOFU users directly. A/B test progressive vs flat.

    Full rollout — All partners
    Based on Tradeit + Winline results, roll out winning combination to WhiteMarket, PaySafe, and future partners.
