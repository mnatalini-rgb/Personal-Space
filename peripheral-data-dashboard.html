Section D: Design Questions to Resolve

1. Setup Volume & Priority

Turkey: 32K records | India: 2.8K

Turkey has 11.4x more data than India. Should we design the hardware display to cater primarily to desktop custom-builds (common in tier 1) or account for large volumes of gaming laptops (Monster/Casper)?

2. Handling Missing Data

~15% Generic / Empty

Users often have "(standart sistem aygıtları)" or empty fields. We should collapse generic inputs to simply "Keyboard" / "Mouse" without a brand rather than exposing the raw OS string.

3. Displaying OEMs

8.2% TR Local OEMs

When a user has a laptop (e.g., Monster), showing "Motherboard: Monster" might be confusing. Better mapping: if System Manufacturer = Motherboard brand, display as "System" instead of "Motherboard".

4. Do we need brand logos?

Top 5 Brands = ~50% volume

It is highly feasible to create logo assets for top Tier 1 brands (Logitech, Razer, SteelSeries, MSI, ASUS), and fallback to clean typography for the long tail.
